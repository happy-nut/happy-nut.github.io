# /refine - 초안을 블로그 글로 다듬기

초안의 핵심 아이디어를 살리면서, 이 블로그만의 문체와 구조로 멋지게 풀어쓴다.

## 사용법
```
/refine @파일경로
/refine (현재 열린 파일)
```

## 이 블로그의 글쓰기 철학

**솔직함**: 꾸미지 않은 개인적 경험과 생각을 담백하게 풀어낸다.
**깊이**: 표면적 설명이 아닌, 왜 그런지에 대한 통찰을 담는다.
**구조**: 독자가 자연스럽게 따라올 수 있는 흐름을 만든다.

## 문체 규칙

### 종결어미
- "~다" 체 사용 (해요체, 합쇼체 사용 금지)
- 예: "생각한다", "느꼈다", "중요하다"

### 강조
- **볼드체**로 핵심 개념/메시지 강조
- 너무 많이 쓰지 않음 (섹션당 1-2개)

### 톤
- 담백하고 직관적 (단, 담백함 ≠ 짧은 문장. 자연스럽게 흐르면서 군더더기 없는 것)
- 과장 없이 솔직하게
- 교훈적이되 설교조는 피함
- 단정적 표현 지양, 여운 있는 개인 견해 톤
- "~지 않을까 싶지만", "~고 본다", "~고 느낀다" 선호
- "~것 같다" 보다 "~라고 생각한다" 선호

### 피해야 할 표현
- "~했습니다", "~입니다" (존댓말)
- "정말로", "진짜로", "매우" 등 과잉 수식어
- "여러분", "독자 여러분" 등 호칭
- 이모지 사용 금지

## 구조 템플릿

```markdown
---
(기존 front matter 유지)
---

[도입부: 질문, 인용, 또는 개인적 맥락으로 시작]
[1-2문단으로 글의 배경과 방향 제시]

## [첫 번째 핵심 개념]

[개념 설명]
[구체적 예시나 경험]
[통찰 또는 시사점]

## [두 번째 핵심 개념]

...

## [마지막 섹션 (보통 실천/적용/전망)]

[구체적인 적용 방안이나 다음 단계]

---

[1-2문단의 깔끔한 마무리]
[핵심 메시지를 한 문장으로 압축]
```

## 변환 과정

**중요: 내용을 먼저 완성한 뒤, 제목을 마지막에 붙인다.**

1. **핵심 추출**: 초안에서 전하고 싶은 핵심 메시지 3-5개 파악
2. **구조 설계**: 메시지들을 논리적 흐름으로 배치
3. **도입부 작성**: 독자의 관심을 끄는 질문이나 맥락 제시
4. **본문 확장**: 각 핵심을 깊이 있게 풀어냄
   - 추상적 개념 → 구체적 예시
   - 주장 → 근거/경험
   - 문제 → 해결책/통찰
5. **마무리 응축**: 글 전체를 관통하는 한 문장 메시지
6. **제목 생성**: 완성된 글의 핵심 메시지에서 제목 추출
7. **메타데이터**: 카테고리, 태그, description 추론
8. **썸네일 이미지**: 글의 첫 번째 이미지를 `image` 속성으로 front matter에 추가

## 확장 기법

### 초안이 단편적일 때
- 숨은 연결고리 찾기: 각 아이디어가 어떻게 연결되는지
- 맥락 추가: 왜 이 생각을 하게 되었는지
- 시사점 도출: 그래서 어떻게 해야 하는지

### 초안이 나열형일 때
- 공통 주제 발견: 나열된 것들을 관통하는 하나의 메시지
- 계층화: 중요도에 따라 재배치
- 스토리화: 시간순/논리순 흐름 만들기

## 분량 가이드

- 최소: 1200자 (짧은 인사이트)
- 적정: 2500-3500자 (일반 포스트)
- 최대: 5000자 (심층 분석)

초안이 짧아도 핵심만 있다면 충분히 확장 가능.
단, 억지로 늘리지 말고 의미 있는 확장만.

## 이미지 처리

초안에 `_posts/` 디렉토리 내 이미지 참조가 있으면 자동으로 정리한다.

### 처리 흐름
1. 포스트 제목과 이미지 주변 맥락 분석
2. 영문 kebab-case로 이미지 이름 자동 생성 (예: `trading-system`, `brain-structure`)
3. `_posts/{이미지}` → `assets/img/posts/{자동생성이름}.{확장자}` 로 이동
4. 포스트 내 이미지 참조 업데이트
5. alt 텍스트도 맥락에 맞게 한글로 자동 생성

### 변환 예시
```
Before: ![img.png](img.png)
After:  ![트레이딩 시스템 화면](/assets/img/posts/trading-system.png)
```

### 규칙
- 이미지 이름: 영문 kebab-case (소문자, 하이픈 구분)
- alt 텍스트: 한글, 간결하게 (맥락에서 자동 추론)
- 사용자 확인 없이 바로 적용

### 다이어그램 및 시각화

계산 과정이나 시스템 흐름을 설명할 때 `mermaid`를 사용한 다이어그램을 추가하면 이해에 도움이 될 수 있다. 하지만 다음 원칙을 따른다.

- **단순성**: 다이어그램은 한눈에 이해하기 쉬워야 한다. 글보다 다이어그램이 더 복잡하게 느껴진다면, 이는 잘못된 시각화다.
- **조화**: 블로그의 전반적인 디자인과 어울리는 스타일을 사용한다. 너무 화려하거나 이질적인 색상 사용은 피한다.
- **대안 고려**: 복잡한 다이어그램 대신, 간단한 마크다운 **표(Table)**를 사용해 정보를 구조화하는 것이 더 효과적일 수 있다. 표는 글의 흐름을 해치지 않으면서 정보를 명확하게 전달하는 좋은 대안이다.

**좋은 예시 (표 활용):**

| 단계 (Step) | 항목 (Item) | 값 (Value) | 계산 (Calculation) |
| :--- | :--- | :--- | :--- |
| 1. 리스크 정의 | 총 계좌 | 100,000,000원 | |
| | **최대 손실액** | **1,000,000원** | `100,000,000원 * 1%` |
| 2. 주당 리스크 | **주당 리스크** | **2,000원** | `50,000원 - 48,000원` |
| 3. 포지션 크기 | **매수 수량** | **500주** | `1,000,000원 / 2,000원`|

- **가운데 정렬**: 마크다운 표를 가운데로 정렬하려면, 표 바로 아랫줄에 `{:.mx-auto}`를 추가한다. 이 방식은 테마의 스타일 클래스(`mx-auto`)를 적용하는 Kramdown 문법이다.
- **캡션 추가**: 표나 이미지 바로 아래에 이탤릭체로 캡션을 추가할 수 있다. 예: `_표 또는 이미지 캡션_`

**가운데 정렬 예시:**
```
| A | B |
|---|---|
| 1 | 2 |
{:.mx-auto}
```

## 실행 후

1. 내용 완성 후 제목 결정
2. 파일명을 `YYYY-MM-DD-{제목}.md` 형식으로 변경 (draft → 실제 제목)
3. 변환된 글을 파일에 저장
4. 이미지 파일 이동 완료
5. 로컬 서버 실행하여 결과 확인 (`bundle exec jekyll s`)

## 예시: 초안 → 완성글

### Before (초안)
```
AI가 발전하면서 개발자도 대체될 것 같다
회사에 의존하면 안된다
스스로 가치를 만들어야 한다
나는 투자를 시작했다
트레이딩 시스템을 만들고 있다
```

### After (완성글)
```markdown
AI가 코드를 작성하는 시대가 왔다.

불과 몇 년 전만 해도 "AI가 개발자를 대체한다"는 말은 먼 미래의 이야기처럼 들렸다.
하지만 지금 나는 매일 AI와 함께 코드를 작성하고, 솔직히 특정 영역에서는
AI가 나보다 낫다는 것을 인정할 수밖에 없다.

## 일자리라는 안전망

...

## 가치를 증명하는 삶

...

## 투자라는 선택

...

---

결국 중요한 건 회사가 나를 필요로 하는지가 아니라,
시장이 나의 가치를 인정하는지다. 그 가치를 어떻게 만들어갈 것인지,
그것이 AI 시대를 살아가는 우리 모두의 숙제가 아닐까.
```

## SEO description 자동 생성

front matter에 `description` 필드 추가. 검색 결과 미리보기에 표시됨.

### 규칙
- 글의 핵심 내용을 1-2문장으로 요약
- 키워드 포함 (AI, 투자, 개발, 트레이딩 등)
- 80-150자 내외
- 클릭을 유도하는 문구

### 예시
```yaml
description: AI가 일자리를 대체하는 시대, 회사에 의존하지 않고 독립적으로 생존하는 방법
```

## 썸네일 이미지 자동 설정

공유 시 표시되는 썸네일 이미지를 설정한다.

### 규칙
- 초안 작성 시 썸네일로 사용하고 싶은 이미지의 alt 텍스트를 `thumbnail`로 지정한다.
  - 예: `![thumbnail](image.png)`
- 변환 과정에서 해당 이미지를 찾아 front matter의 `image` 속성으로 설정한다.
- `thumbnail`로 지정된 이미지가 없는 경우, `image` 속성을 추가하지 않으며 기본 썸네일 이미지(`/assets/img/default-thumbnail.png`)가 사용된다.

### 예시
```yaml
image: /assets/img/posts/my-thumbnail-image.png
```

## 카테고리/태그 자동 추론

글 내용을 분석하여 가장 적절한 카테고리와 태그를 자동 선택한다. **이때, 모든 영문은 소문자를 사용하고, 띄어쓰기는 하이픈(-)으로 연결하는 `kebab-case`를 엄격히 준수하여 빌드 오류를 방지한다.**

### 카테고리 매핑

| 주제 키워드 | 카테고리 |
|-------------|----------|
| 회고, 월간, 분기 | `[retrospect]` |
| Kotlin, 코루틴, Spring | `[engineering, kotlin]` |
| JVM, GC, 메모리 | `[engineering, jvm]` |
| 아키텍처, 설계, 도메인, 클린 | `[engineering, architecture]` |
| 운영, 배포, graceful, shutdown | `[engineering, operation]` |
| 소스코드, 분석, 뜯어보기 | `[engineering, analysis]` |
| Redis, 캐시, 분산 | `[engineering, redis]` |
| Kafka, 메시징, 이벤트 | `[engineering, kafka]` |
| AI, LLM, 파인튜닝, GPT | `[ai, llm]` |
| AI + 경제, 일자리, 자본 | `[ai, economy]` |
| 의사결정, 비즈니스, 전략 | `[business, decision]` |
| 리더십, 조직, 권력, 팀 | `[leadership, psychology]` |
| 사고, 문제해결, 구조적 | `[thinking, problem-solving]` |
| 투자, 자본, 금융, 소득 | `[finance, investment]` |
| 감정, 심리, 자기계발 | `[psychology, self-development]` |
| 삶, 육아, 부모, 가족 | `[life, parenting]` |

### 태그 생성 규칙
- 모든 태그는 **영문 소문자**와 **케밥-케이스(kebab-case)**를 사용한다.
- 글 제목과 본문에서 핵심 키워드 3-7개 추출한다.
- 구체적인 기술명/개념명을 우선으로 한다. (예: `coroutine`, `graceful-shutdown`, `esbi`)

### 태그 예시
| 글 주제 | 태그 |
|---------|------|
| 코루틴 비동기 동작 | `[kotlin, coroutine, async, suspend, dispatcher]` |
| 트레이딩 시스템 | `[trading, system, investment, automation]` |
| 감정 다루기 | `[emotion, psychology, brain, prefrontal-cortex]` |
| 의사결정 프레임워크 | `[decision-making, framework, one-way, two-way]` |

## 주의사항

- 초안의 핵심 메시지를 왜곡하지 않는다
- **없는 경험이나 사실을 지어내지 않는다**
  - 특히 대화 내용, 행동 순서, 구체적 질문 등은 초안에 없으면 추가 금지
  - 확장은 통찰, 맥락, 배경 지식으로만 (사용자의 실제 경험 창작 금지)
- **개념 설명 시 예시와 비유 적극 활용**
  - 숫자, 상황, 비유 등으로 이해도를 높임
  - 추상적 설명만 하지 않고 "예를 들어~" 형태로 구체화
  - 복잡한 개념일수록 일상적 비유로 풀어줌
- 작성자의 목소리를 유지한다 (너무 다른 사람처럼 쓰지 않음)
- 이미지에 캡션 추가 (이미지 바로 아래 `_캡션 내용_` 형식)
