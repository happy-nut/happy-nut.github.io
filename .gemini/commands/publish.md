# /publish - 커밋 & 푸시

## 사용법
```
/publish [메시지]
```

## 실행 흐름
1. **가이드 개선**: 현재까지의 작업 피드백을 검토하고, `refine.md` 와 같은 가이드 파일에 개선점을 반영한다.
2. **로컬 빌드 검증**: `bundle exec jekyll build`를 실행하여 빌드 오류나 `Conflict`와 같은 경고가 없는지 확인한다. 문제가 있다면 푸시하기 전에 먼저 해결한다.
3. `git status`로 변경사항 확인
4. 변경사항 분석하여 커밋 메시지 자동 생성 (메시지 미입력 시)
5. `git add .` 후 커밋
6. `git push`로 원격 저장소에 푸시

## 커밋 메시지 규칙
- 한글로 작성
- 단촐하게 핵심만
- Co-Authored-By, Generated by 등 불필요한 내용 제외

## 자동 메시지 예시
| 변경 내용 | 메시지 |
|-----------|--------|
| _posts/ 파일 추가 | `new post` |
| til/ 파일 추가 | `new til` |
| koongya/ 파일 추가 | `new diary` |
| 파일 수정 | `update {파일명}` |
| 여러 파일 수정 | `update files` |
| 설정 변경 | `update config` |

## 예시
```
/publish
→ 변경사항 분석 후 자동 메시지 생성, 커밋 & 푸시

/publish "about 페이지 수정"
→ 지정한 메시지로 커밋 & 푸시
```
